# Changelog

## v1.0.0 — 2026-07-13

Initial release. Extracted as the generic design canon from two worked
implementations (a library-scale skill audit and an N=1 knowledge-extraction
gate) that had each embedded the same evaluation discipline independently:

- Three principles: binary checks as evidence / named holistic verdict / no
  score aggregation
- Verdict pressure-test (refutation-oriented atomic questions, same-question-set
  re-judge, once only)
- Scale asymmetry guidance (unconditional questions at N=1, candidates-only at
  library scale)
- Verification-ownership section grounded in a 2026-07 controlled comparison
  (single-context all-Keep vs fresh-context batches surfacing 12/73 non-passing)
- Portable judge prompt template + JSON output schema
